# Personnel & Organization SPA

Привет! Это моё тестовое задание.
---

## Что это вообще такое

SPA-приложение для ведения справочников персонала и организаций. Можно добавлять организации,
добавлять к ним сотрудников с фотографиями, искать по разным полям и смотреть журнал всех
действий. Ещё есть кнопка генерации тестовых данных — она создаёт 10 организаций и 1000
сотрудников одним нажатием.

---

## Стек

**Бэкенд:**
- ASP.NET Core 9 Web API
- ADO.NET (SqlConnection, SqlCommand, SqlDataReader) — без ORM
- MS SQL Server 2022

**Фронтенд:**
- React 18 + TypeScript
- Vite
- React Router v6
- Чистый CSS

**Инфраструктура:**
- Docker + docker-compose

---

## Как запустить

Нужен Docker Desktop. Больше ничего устанавливать не надо.

```bash
docker compose up -d --build
```

После этого открыть в браузере: **http://localhost:8080**

При первом запуске SQL Server стартует секунд 30-40, приложение подождёт его автоматически
(там есть healthcheck). Если `app` контейнер упал — просто подождите и перезапустите:

```bash
docker compose restart app
```

База данных и все таблицы создаются автоматически при первом запуске — никаких скриптов
запускать не нужно.

## API эндпоинты

| Метод  | URL                          | Описание                              |
|--------|------------------------------|---------------------------------------|
| GET    | /api/organizations           | Список всех организаций               |
| POST   | /api/organizations           | Создать организацию                   |
| PUT    | /api/organizations/{id}      | Обновить организацию                  |
| DELETE | /api/organizations/{id}      | Удалить организацию (каскадно)        |
| GET    | /api/employees               | Поиск сотрудников с фильтрами         |
| POST   | /api/employees               | Создать сотрудника (multipart)        |
| PUT    | /api/employees/{id}          | Обновить сотрудника                   |
| DELETE | /api/employees/{id}          | Удалить сотрудника                    |
| GET    | /api/employees/{id}/photo    | Получить фото сотрудника              |
| GET    | /api/actionlogs              | Журнал действий с пагинацией          |
| POST   | /api/seed                    | Сгенерировать тестовые данные         |

## Что реализовано

- [x] CRUD сотрудников с загрузкой фото (хранится в БД как VARBINARY)
- [x] Поиск сотрудников по 7 полям одновременно (LIKE + AND)
- [x] Журнал всех действий (Create/Update/Delete/SeedData)
- [x] Генерация 1000 тестовых записей одной кнопкой(10x100)
- [x] Автоматическое создание БД и таблиц при первом запуске
- [x] Docker + docker-compose с healthcheck
